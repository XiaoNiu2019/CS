OUTPUT
################################################################################################
An XML file named (results_inputfile) is generated (in the same repository with the main 
program) with the following computations :
	1) End to end delays bounds for each flow
	2) Loads of each link 
################################################################################################
Critics of the program
################################################################################################
The program gives good results for all loads computations .

For the delay computations, results are good for the following use-cases :
	EE.xml 
	ESE.xml
	ESSE.xml
	ES2E_M.xml
	ESE_F3.xml
	3ESE.xml

However, for use-cases which involve multiple sources and targets (STAR_3.xml, ISAE_TEST_1.xml,
ISAE_TEST_2.xml, AFDX.xml), the algorithm is not good (which correspond to lines 502-575 in the 
code) which gives incorrect outputs... (A point to be corrected later when getting a chance)

Concerning the structure of the code, in general it is working well and the computational time 
is not long. However a more proper way can always be done by defining classes andmethods in 
order to improve the program's performance.
